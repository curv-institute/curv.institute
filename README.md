# CURV Institute Website

Static site built with Hugo, hosted on GitHub Pages at https://curv.institute/

## Structure

```
├── content/
│   ├── _index.md           # Homepage (content in layouts/index.html)
│   ├── blog/               # Blog posts
│   │   └── _index.md
│   └── papers/             # Paper pages (metadata + abstract)
│       └── _index.md
├── layouts/                # Hugo templates
│   ├── _default/
│   │   └── baseof.html
│   ├── index.html          # Homepage template
│   ├── blog/
│   │   ├── list.html
│   │   └── single.html
│   └── papers/
│       ├── list.html
│       └── single.html
├── papers/                 # TeX source files (compiled to PDF on build)
├── static/
│   ├── css/style.css
│   └── papers/             # Compiled PDFs (generated by CI)
└── hugo.toml               # Hugo configuration
```

## Adding a Blog Post

Create a new file in `content/blog/` with frontmatter:

```markdown
---
title: "Your Post Title"
date: 2026-01-12
author: "Author Name"
summary: "A brief summary for the listing page."
---

Your content here...
```

## Adding a Paper

1. **Add the TeX source** in `papers/your-paper.tex`

2. **Create the Hugo page** in `content/papers/your-paper.md`:

```markdown
---
title: "Paper Title"
date: 2026-01-12
authors: "Author One, Author Two"
abstract: "Your paper abstract goes here."
pdf: "/papers/your-paper.pdf"
---

Additional content, notes, or supplementary materials...
```

The CI workflow will automatically compile all `.tex` files in `papers/` to PDFs in `static/papers/`.

## Local Development

Install [Hugo](https://gohugo.io/installation/) and run:

```bash
hugo server -D
```

## Deployment

Push to `main` branch. GitHub Actions will:
1. Compile all TeX papers to PDF
2. Build the Hugo site
3. Deploy to GitHub Pages

## GitHub Pages Setup

In repository Settings > Pages:
- Source: GitHub Actions
- Custom domain: curv.institute (if using custom domain)
